# Vellastra 模块深度挖掘建议

> 基于当前代码现状，针对各模块可做的深度业务逻辑优化建议

---

## 一、auth 模块（认证鉴权）— 当前已较完善

### 1.1 已实现的深度功能
- ✅ JWT 双字段（userId + username + roles）签发/验签/解析
- ✅ Token 黑名单（登出失效）
- ✅ 登录失败锁定（5次/15分钟）
- ✅ 登录审计日志（t_login_log）
- ✅ BCrypt 密码加密 + 强度校验
- ✅ RBAC 权限注解（@RequirePermission + AOP 切面）
- ✅ 网关鉴权过滤器（白名单配置化、请求头注入）

### 1.2 建议追加

| 优先级 | 建议 | 说明 | 预估 |
|--------|------|------|------|
| P1 | **refresh_token / access_token 双Token机制** | 短期 access_token（15分钟）+ 长期 refresh_token（7天），refresh_token 仅用于换新 access_token，降低泄漏风险 | 2h |
| P1 | **权限缓存（Redis）** | 用户登录后将权限列表写入 Redis，AOP 切面从 Redis 查权限，避免每次请求查 DB | 2h |
| P2 | **OAuth2 第三方登录** | GitHub / Google 登录，绑定已有账号或自动注册 | 4h |
| P2 | **操作日志注解** | 自定义 `@OperationLog` 注解 + AOP，自动记录接口调用日志到 t_operation_log | 2h |

## 二、user 模块（用户管理）— 当前 CRUD 已完整

### 2.1 已实现的功能
- ✅ 分页查询（keyword 模糊搜索 + status 筛选）
- ✅ 新增用户（BCrypt 加密 + 参数校验）
- ✅ 编辑用户
- ✅ 逻辑删除
- ✅ 启用/禁用
- ✅ 管理员重置密码
- ✅ 用户自助修改密码

### 2.2 建议追加

| 优先级 | 建议 | 说明 | 预估 |
|--------|------|------|------|
| **P0** | **Feign 远程调用** | auth 模块通过 Feign 调用 user 模块获取用户信息（T2.5） | 2h |
| P1 | **用户分组/部门** | 增加 t_dept 表，用户归属部门，支持按部门筛选 | 3h |
| P1 | **用户导出 Excel** | EasyExcel 导出用户列表，支持按条件筛选后导出 | 2h |
| P1 | **批量导入用户** | Excel 批量导入，模板下载 + 校验结果回显 | 3h |
| P2 | **用户登录IP地理信息** | 登录日志记录 IP 的地理位置（通过 ip2region 库） | 1h |
| P2 | **用户操作日志** | 记录用户增删改等敏感操作的操作人、操作时间、操作内容 | 2h |
| P2 | **用户注册审核** | 新用户注册后需管理员审核才能登录 | 3h |

## 建议执行顺序

```
Phase 2 收尾：
  T2.5 Feign 远程调用（auth→user）      ← 下一步

Phase 3 文章核心：
  T3.1~T3.6 文章CRUD/状态机/搜索/置顶/浏览/点赞

Phase 4 辅助功能：
  T4.1 分类树 CRUD
  T4.2 标签管理
  T4.3 文件上传
  T4.4 文章与分类/标签联动

Phase 5 评论互动：
  T5.1~T5.4 评论/回复/审核/列表树

持续优化（可穿插）：
  - 操作日志注解
  - Redis 缓存
  - 接口文档
  - 单元测试
```
