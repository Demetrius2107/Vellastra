# V3 — UGC 知识社区

> 目标：类似知乎的 UGC 社区平台。
> 用户规模：万人以上。
> 原则：在 V2 CMS 基础上构建社区生态。

---

## 一、V3 新增的能力

```
V2（通用 CMS）                  V3（UGC 社区）
─────────────────              ─────────────────
文章管理           ───→  用户生成内容（UGC）
评论系统           ───→  问答 + 点赞 + 收藏
用户管理           ───→  关注 + 粉丝 + 等级
                          推荐 Feed 流
                          热度排行榜
                          问答系统（类似知乎）
                          通知系统（点赞/评论/关注）
                          举报与审核
                          运营后台
                          反垃圾系统
                          搜索（Elasticsearch）
```

## 二、新增模块

### 2.1 Feed 推荐流

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /feed/recommend | 个性化推荐（登录用户）|
| GET | /feed/latest | 最新内容流 |
| GET | /feed/following | 关注的人的内容 |
| GET | /feed/hot | 热门内容 |

**推荐算法（渐进式）：**
1. V3.0: 简单的时间线 + 热度排序
2. V3.1: 基于标签的粗推荐（用户关注什么标签 → 推荐什么）
3. V3.2: 协同过滤（用户 A 和 B 喜欢相似内容 → 互相推荐）

### 2.2 问答系统

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /question | 提问题 |
| GET | /question/{id} | 查看问题（含回答列表）|
| GET | /question | 问题列表（分页+筛选）|
| POST | /answer | 回答问题 |
| PUT | /answer/{id} | 编辑回答 |
| POST | /answer/{id}/accept | 采纳回答 |
| POST | /answer/{id}/vote | 投票（赞同/反对）|

### 2.3 互动与通知

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /interaction/like | 点赞/取消 |
| POST | /interaction/follow | 关注用户/话题 |
| POST | /interaction/bookmark | 收藏文章/回答 |
| GET | /notification | 通知列表 |
| PATCH | /notification/{id}/read | 标记已读 |

**通知类型：**
- 有人赞了你的文章
- 有人回复了你的评论
- 有人关注了你
- 有人回答了你的问题
- 你的回答被采纳

### 2.4 热度排行榜

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /rank/daily | 日榜 |
| GET | /rank/weekly | 周榜 |
| GET | /rank/monthly | 月榜 |

**热度算法（Hacker News 风格）：**
```
score = (点赞数 - 踩数) / (time)^1.5
```
后续可替换为更复杂的算法（考虑阅读时长、分享数等）

### 2.5 内容审核与风控

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /audit/pending | 待审核列表 |
| POST | /audit/approve | 通过 |
| POST | /audit/reject | 拒绝（含原因）|
| POST | /audit/report | 用户举报内容 |
| GET | /audit/reports | 举报列表 |

**审核机制：**
1. 新用户前 5 篇文章需人工审核
2. 信用分达标后自动过审
3. 敏感词命中自动拦截
4. 举报触发重新审核

### 2.6 用户体系升级

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /user/{id}/profile | 用户主页（文章/回答/收藏）|
| GET | /user/{id}/followers | 粉丝列表 |
| GET | /user/{id}/following | 关注列表 |
| PATCH | /user/level | 用户等级/经验值 |
| GET | /user/{id}/stats | 用户统计（获赞/阅读量）|

**等级体系（参考知乎/掘金）：**
- Lv1: 注册用户 → 可以发文、评论
- Lv2: 发布 5 篇以上 → 可以创建专栏
- Lv3: 100 赞以上 → 内容优先推荐
- Lv4: 1000 赞以上 → 创作者认证
- Lv5: 10000 赞以上 → 社区 KOL 标识

### 2.7 搜索

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /search | 全局搜索（文章/问题/用户）|
| GET | /search/suggestion | 搜索建议（前缀匹配）|

**技术选型：** Elasticsearch
- 文章/问题的标题 + 正文建索引
- 用户昵称 + 简介建索引
- 搜索建议用 completion suggester

## 四、V3 新增基础设施

| 组件 | 用途 |
|------|------|
| Elasticsearch | 全文搜索 + 搜索建议 |
| RabbitMQ / RocketMQ | 异步通知、Feed 分发、构建触发 |
| Redis Cluster | 排行榜、Feed 缓存、点赞去重、限流 |
| OSS (阿里云/MinIO) | 图片/文件存储，分离业务服务器 |
| CDN | 静态资源加速 |
| ELK | 日志分析、业务监控 |
| Prometheus + Grafana | 服务监控、告警 |
| XXL-JOB / Quartz | 定时任务（热度结算、数据清理）|
